## What Is the Process-Oriented Approach to Software Quality?
The process-oriented approach focuses on how software is developed, rather than only evaluating the final product.


## Core Philosophy
| Product-Oriented      | Process-Oriented                |
| --------------------- | ------------------------------- |
| Focus on final output | Focus on development activities |
| Testing at the end    | Quality built in from start     |
| Fix defects later     | Prevent defects early           |
| Reactive              | Preventive                      |


## Key Principles of the Process-Oriented Approach
#### 1. Quality Is Built In, Not Inspected In
Quality activities start at:
- Requirements
- Design
- Coding
- Testing
- Not just during final testing

📌 Example:
Requirements reviews to prevent ambiguity before coding starts.


#### 2. Standardized and Defined Processes
Organizations follow documented procedures for:
- Requirements management
- Design reviews
- Coding standards
- Testing strategy
- Change management

📌 Example standards:
- ISO/IEC 15504
- CMMI


#### 3. Continuous Improvement
Processes are:
- Measured
- Analyzed
- Improved over time

This commonly follows the PDCA Cycle:
- Plan – Define processes
- Do – Execute processes
- Check – Audit & measure
- Act – Improve processes


#### 4. Prevention Over Detection
| Prevention          | Detection      |
| ------------------- | -------------- |
| Training developers | Bug fixing     |
| Code reviews        | Debugging      |
| Design validation   | Patch releases |

📌 Fixing a defect after release costs much more than preventing it early.


#### 5. Process Measurement & Metrics
Quality is tracked using process metrics, not just defects. Examples:
- Defect Injection Rate
- Defect Removal Efficiency (DRE)
- Review effectiveness
- Test coverage
- Cycle time


## Common Process-Oriented Quality Models
1. CMMI (Capability Maturity Model Integration)
2. ISO 9001


## Agile & DevOps (Modern View)
Even Agile is process-oriented:
- Sprint planning
- Daily stand-ups
- Retrospectives
- CI/CD pipelines

📌 Agile emphasizes adaptive processes, not no process.


## Why Process-Oriented Quality Matters
| Stakeholder  | Benefit               |
| ------------ | --------------------- |
| Management   | Predictable delivery  |
| Developers   | Clear guidelines      |
| Testers      | Early involvement     |
| Customers    | Consistent quality    |
| Organization | Reduced rework & cost |


## Mapping to SQA Activities
| SDLC Phase   | Process-Oriented Activity |
| ------------ | ------------------------- |
| Requirements | Requirement inspections   |
| Design       | Design walkthroughs       |
| Coding       | Coding standards, reviews |
| Testing      | Test planning, metrics    |
| Deployment   | Release procedures        |
| Maintenance  | Change management         |


## Process-Oriented vs Product-Oriented
| Aspect   | Process-Oriented      | Product-Oriented       |
| -------- | --------------------- | ---------------------- |
| Focus    | How software is built | What software delivers |
| Timing   | Throughout lifecycle  | Mostly at the end      |
| Goal     | Prevent defects       | Detect defects         |
| Examples | CMMI, ISO/IEC 15504   | ISO/IEC 25010          |
