## What Is the Product-Oriented Approach to Software Quality?
The product-oriented approach focuses on the quality of the software product itself — its features, performance, reliability, usability, and other observable characteristics.


## Core Focus of Product-Oriented Quality
| Focus Area  | Description                     |
| ----------- | ------------------------------- |
| Output      | Final software system           |
| Measurement | Quality attributes & metrics    |
| Evaluation  | Testing, inspection, validation |
| Timing      | Mostly after development        |
| Nature      | Reactive (detect defects)       |


## How Product-Oriented Quality Is Evaluated
| Technique           | Purpose                   |
| ------------------- | ------------------------- |
| Functional testing  | Verify requirements       |
| Performance testing | Measure speed & load      |
| Usability testing   | Evaluate user experience  |
| Security testing    | Identify vulnerabilities  |
| Reliability testing | Check stability           |
| Acceptance testing  | Confirm stakeholder needs |
📌 Quality is validated through testing & inspection.


## Limitations of Product-Oriented Approach
| Limitation            | Explanation                          |
| --------------------- | ------------------------------------ |
| Late defect discovery | Issues found after development       |
| High rework cost      | Fixing late defects is expensive     |
| No process insight    | Does not explain *why* defects occur |
| Quality inconsistency | Same process not guaranteed          |
📌 That’s why modern SQA combines product + process approaches.
