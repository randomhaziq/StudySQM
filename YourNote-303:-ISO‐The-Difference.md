## ISO/IEC 15504 vs ISO/IEC 25010
Both ISO/IEC 15504 and ISO/IEC 25010 are international standards related to software quality—but they focus on different dimensions:
- ISO/IEC 15504 focuses on the quality of software processes.
- ISO/IEC 25010 focuses on the quality of software products.

## Purpose
### ISO/IEC 15504 (SPICE) – Process Assessment Standard
- Evaluates software development processes.
- Measures process capability and process maturity.
- Helps organisations improve their software development methods.

### ISO/IEC 25010 – Product Quality Model
- Evaluates the quality of a software product.
- Defines product quality characteristics and quality-in-use characteristics.
- Used to assess how well the system performs for the end user.

## Focus Area
| Standard          | Focus                                                 |
| ----------------- | ----------------------------------------------------- |
| **ISO/IEC 15504** | How well processes are performed (process capability) |
| **ISO/IEC 25010** | How good the software product is (product quality)    |


## What They Measure
#### ISO/IEC 15504 Measures:
- Process performance
- Management practices
- Resource allocation
- Consistency of process execution
- Capability levels (0 to 5)
#### Example:
- How well does the development team manage requirements, testing, configuration, and maintenance?

#### ISO/IEC 25010 Measures:
- Functional suitability
- Reliability
- Usability
- Maintainability
- Performance efficiency
- Security
- Compatibility
- Portability
#### Example:
- How correct, fast, secure, usable, and stable is the software product?

## Structure
#### ISO/IEC 15504
Uses Capability Levels 0–5:  
0. Incomplete
1. Performed
2. Managed
3. Established
4. Predictable
5. Optimizing

#### ISO/IEC 25010
Uses 8 Product Quality characteristics:
1. Functional suitability
2. Performance efficiency
3. Compatibility
4. Usability
5. Reliability
6. Security
7. Maintainability
8. Portability
- Also includes Quality-in-Use Characteristics:
  - Effectiveness
  - Efficiency
  - Satisfaction
  - Freedom from risk
  - Context coverage

## Usage / Application
| ISO Standard      | Who Uses It?                                    | For What Purpose?                                                       |
| ----------------- | ----------------------------------------------- | ----------------------------------------------------------------------- |
| **ISO/IEC 15504** | Organizations, process engineers                | Improving software development processes; audits; CMMI-like assessments |
| **ISO/IEC 25010** | Testers, quality analysts, developers, UX teams | Defining and measuring software product quality                         |


## Example Scenario
#### ISO/IEC 15504 Example:
An IT company wants to improve its testing process. An assessor evaluates whether the testing process is:
- repeatable
- documented
- controlled
- measurable

#### ISO/IEC 25010 Example:
A mobile banking app is evaluated on:
- security
- usability
- reliability
- performance

## Summary
| Aspect              | **ISO/IEC 15504 (SPICE)**  | **ISO/IEC 25010**                 |
| ------------------- | -------------------------- | --------------------------------- |
| **Focus**           | Software processes         | Software product quality          |
| **Goal**            | Improve process capability | Improve product quality           |
| **Assessment Type** | Process assessment         | Product evaluation                |
| **Output**          | Capability level (0–5)     | Quality score for characteristics |
| **Users**           | Organizations, auditors    | Developers, QA, testers           |
| **Scope**           | How software is built      | The quality of the final software |
