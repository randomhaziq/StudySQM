# Software Quality Control

---

## Definition
Software Quality Control is the set of activities that focus on detecting defects in the software product through structured testing, inspections, and reviews to ensure it meets quality requirements.

---

## Difference Between Quality Control and Quality Assurance
| **Quality Control (QC)**       | **Quality Assurance (QA)**                   |
| ------------------------------ | -------------------------------------------- |
| Product-focused                | Process-focused                              |
| Detects defects                | Prevents defects                             |
| Conducted after implementation | Conducted throughout the development process |
| Testing, inspections, reviews  | Process definition, standards, audits        |

---

## Goal

To provide students with an understanding of Software Quality Control (SQC), its importance in ensuring software quality throughout the software development lifecycle, the use of quality tools for defect analysis and process improvement, and the application of the Reliability model for analyzing software defect distribution and reliability growth.

---

## 🎯 Learning Objectives

Students should be able to:

1. Explain the concept of Software Quality Control and its role in the software lifecycle.
2. Identify and apply basic quality tools for defect analysis and process improvement.
3. Interpret and apply the Reliability model in software defect distribution.
